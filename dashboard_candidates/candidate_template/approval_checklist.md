# Shadow Dashboard Candidate Approval Checklist

## Research Evidence

- [ ] Source hypotheses are listed.
- [ ] Exact feature columns were verified against the feature inventory.
- [ ] No live sentiment or headline feature is used for historical validation unless point-in-time archives exist.
- [ ] Alert days are reported.
- [ ] Alert burden is reported.
- [ ] Base-rate lift is reported.
- [ ] Event coverage is reported.
- [ ] Positive-lift folds are reported.
- [ ] Median lead time is reported.
- [ ] False calm behavior is documented.
- [ ] Failure modes are documented.

## Shadow Candidate Review

- [ ] Candidate rules are deterministic and pre-declared.
- [ ] No threshold optimization was used.
- [ ] Shadow report exists.
- [ ] Comparison versus current dashboard exists.
- [ ] Candidate is labeled as research-only.

## Production Approval

- [ ] Human reviewer has approved production implementation.
- [ ] Production files allowed to change are listed by exact path.
- [ ] Rollback or revert plan exists.
- [ ] Implementation plan exists.
- [ ] Approval status file is updated.
